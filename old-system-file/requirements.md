## 五、库存管理说明 ✅

**库存定义：** 库存 = 收货数量 - 发货数量

**用途：**
- 查看待发货差额（还有多少产品需要安排发货）
- 辅助排产决策（差额大就需要排产）
- 对账依据

**特点：**
- 不需要库存预警
- 一般收货发货数额相等时，库存归零
- 是"待处理量"而非"仓库存储量"

```
收货 100 件 → 库存 +100（待发货 100）
发货 80 件  → 库存 = 20（待发货还剩 20）
发货 20 件  → 库存 = 0（全部发完）
```

## 七、业务规则（已确认 ✅）

### 7.1 价格记忆规则

**录入收货单时的单价处理：**

```
1. 新增收货明细 → 选择物料 → 自动带出该物料的"最近单价"
2. 用户可修改单价
3. 保存收货单后 → 更新物料的默认单价为本次单价
4. 下次再选这个物料 → 带出更新后的单价
```

**示例：**
```
物料A 当前默认单价：10元
├─ 收货单1：选物料A → 带出10元 → 修改为12元 → 保存
│                                      ↓
│                           物料A默认单价更新为12元
├─ 收货单2：选物料A → 带出12元 → 不修改 → 保存
├─ 收货单3：选物料A → 带出12元 → 修改为11元 → 保存
│                                      ↓
│                           物料A默认单价更新为11元
```

**实现要点：**
- 物料档案表需要 `default_price`（默认单价）字段
- 收货单保存时，更新对应物料的 `default_price`
- 收货明细选择物料时，前端调用接口获取默认单价

### 7.2 单据打印规则 ✅

**打印范围：** 所有单据都支持打印

| 单据类型 | 打印场景 |
|---------|---------|
| 收货单 | 入库凭证 |
| 发货单 | 送货单（给客户） |
| 排产单 | 生产任务单 |
| 返工单 | 返工处理单 |
| 收款单 | 收款凭证 |
| 对账单 | 月度对账单（给客户） |

**打印格式：**
- **纸张大小：** A5 横向（210mm × 148mm）
- **高度自适应：** 内容多时自动撑高
- **自动分页：** 内容超出单页时自动分页

**技术实现建议：**
- 前端使用打印插件（如 `vue-print-nb` 或 `lodop`）
- CSS `@media print` 控制打印样式
- 使用 `page-break-after` 控制分页

```css
/* 打印样式示例 */
@media print {
  @page {
    size: A5 landscape;  /* A5横向 */
    margin: 10mm;
  }
  .page-break {
    page-break-after: always;  /* 自动分页 */
  }
}
```

### 导入需求

老系统数据已导出为 Excel 文件，新系统需要支持**表格数据导入**功能。

### 待导入数据

| 数据 | 文件 | 数据量 | 导入优先级 |

| 物料档案 | 物料档案.xls | 23,166 条 | 高 |

| 收货单 | 收货单.xls | 65,536 条 | 中 |
| 对账单 | 对账单.xls | 122 条 | 低 |

### 导入功能设计

**功能入口：** 系统管理 → 数据导入

**导入流程：**
```
1. 选择导入类型（客户/物料/工艺/收货单/排产单）
2. 下载导入模板（Excel格式）
3. 按模板格式整理数据
4. 上传 Excel 文件
5. 系统校验数据（必填项、格式、重复）
6. 预览导入结果（成功X条，失败Y条）
7. 确认导入
8. 生成导入日志（记录成功/失败明细）
```

**导入特性：**
- 支持 `.xls` / `.xlsx` 格式
- 字段映射：老系统字段 → 新系统字段
- 数据校验：必填、格式、唯一性
- 错误提示：行号 + 字段 + 错误原因
- 支持覆盖/跳过重复数据
- 导入进度显示（大数据量时）

**技术实现：**
- 后端：EasyExcel 或 Apache POI
- 分批读取（每次1000条），避免内存溢出
- 异步导入 + 进度通知（大数据量时）

## 十四、库存优化设计 ✅

### 老系统问题

```
❌ 老系统：没有库存表
   每次查询库存 = SUM(收货数量) - SUM(发货数量)
   
   问题：
   - 查询慢（6.5万条收货单实时统计）
   - 无法快速知道当前待发货量
   - 对账时计算复杂
```

### 新系统优化

```
✅ 新系统：独立库存表 + 实时更新

【库存表设计思路】
┌──────────────────────────────────────────────────────┐
│  物料ID  │  客户ID  │  当前库存  │  最后更新时间     │
└──────────────────────────────────────────────────────┘
      ↓           ↓           ↓
   物料维度    客户维度     实时数量

【库存更新触发】
收货单保存 → 库存 +收货数量
发货单保存 → 库存 -发货数量
返工单保存 → 库存 +返工数量（回收入库）
```

### 历史数据导入策略

```
【导入顺序】
1. 基础数据：客户 → 工艺 → 物料
2. 业务单据：收货单 → 发货单（从收货单中提取）
3. 库存初始化：导入完成后，自动计算初始库存

【库存初始化逻辑】
INSERT INTO inventory (material_id, customer_id, quantity)
SELECT 
    material_id,
    customer_id,
    SUM(receive_qty) - SUM(ship_qty) as quantity
FROM (
    SELECT material_id, customer_id, quantity as receive_qty, 0 as ship_qty FROM receive_orders
    UNION ALL
    SELECT material_id, customer_id, 0 as receive_qty, quantity as ship_qty FROM ship_orders
) t
GROUP BY material_id, customer_id
```

### 库存表设计

```sql
CREATE TABLE inventory (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    material_id BIGINT NOT NULL COMMENT '物料ID',
    customer_id BIGINT NOT NULL COMMENT '客户ID',
    quantity DECIMAL(10,2) NOT NULL DEFAULT 0 COMMENT '当前库存',
    last_receive_time DATETIME COMMENT '最后收货时间',
    last_ship_time DATETIME COMMENT '最后发货时间',
    created_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_material_customer (material_id, customer_id)
) COMMENT '库存表';
```

### 库存流水表（可选）

```sql
CREATE TABLE inventory_log (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    material_id BIGINT NOT NULL,
    customer_id BIGINT NOT NULL,
    change_type TINYINT NOT NULL COMMENT '1收货 2发货 3返工',
    change_qty DECIMAL(10,2) NOT NULL COMMENT '变动数量（正负）',
    before_qty DECIMAL(10,2) COMMENT '变动前库存',
    after_qty DECIMAL(10,2) COMMENT '变动后库存',
    order_type VARCHAR(20) COMMENT '单据类型',
    order_id BIGINT COMMENT '单据ID',
    created_time DATETIME DEFAULT CURRENT_TIMESTAMP
) COMMENT '库存变动流水';
```
